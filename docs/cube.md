## 有哪些重要的概念？
* 模视变换
* 投影变换
* Uniform 变量
* 索引形式的绘制

### 模视变换
如果想从不同方向观察场景，那么既可以变动场景中每个点的坐标，也可以变换「摄像机的位置」。模视变换矩阵更接近于后者，我们可以通过 translate / rotate 等常见的变换方式来操纵这一变换矩阵。

### 投影变换
将三维空间的点「拍扁」到二维空间，即所谓的透视变换。它定义了一个指定了视角、远近范围与宽高比的「视见体」，在这个区域外的点会被丢弃。

> Model View Matrix 与 Projection Matrix 的更多信息，参见图形学的基础知识。

### Uniform 变量
要想对所有的顶点都与某个固定的矩阵相乘，或者是指定一些全局固定的颜色等参数时，可以在着色器中使用 uniform 类型的变量，它不像 attribute 数组那样逐顶点执行，而是更好理解的一个「全局变量」。

### 索引形式的绘制
`drawArrays` 是直接将一整段连续的缓冲区中数据逐个绘制的。对于存在大量重复共享、重复的顶点的情形，`drawElements` API 支持先从索引 buffer 中取出每次想绘制的顶点 buffer 的索引值，再用这个索引值读取顶点 buffer 数组的相应位置，从而复用一些顶点数据。


## 流程大致是什么样的？
* 编译 shader
* 初始化 buffer，包括顶点位置、顶点颜色与顶点索引
* 定义模视变换与投影变换矩阵为 uniform 变量
* 绘制一帧
